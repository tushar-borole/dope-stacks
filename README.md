<h1 id="dope-stacks">dope-stacks</h1>
<p>This repo contain awsome open source stacks</p>
<h2 id="frontend">Frontend</h2>
<ul>
<li><a href="https://reactjs.org/">React</a> : React makes it painless to create<br>
interactive UIs</li>
<li><a href="https://vuejs.org/">Vuejs</a> : Vue.js is an open-source JavaScript framework for building user interfaces and single-page applications.</li>
<li><a href="https://facebook.github.io/react-native">react-native</a> : Build native mobile apps using JavaScript and React</li>
<li><a href="https://flutter.dev/">Flutter</a> : Flutter is an open-source mobile application development framework</li>
<li><a href="https://svelte.dev/">svelte</a> : Instead of using techniques like virtual DOM diffing, Svelte writes code that surgically updates the DOM when the state of your app changes.</li>
<li><a href="https://www.polymer-project.org/">Polymer Project.org/</a> : Web component form web.</li>
<li><a href="https://www.typescriptlang.org/">Typescript</a> : TypeScript is a typed superset of JavaScript that compiles to plain JavaScript.</li>
</ul>
<h2 id="database">Database</h2>
<ul>
<li><a href="https://prestodb.io/">PrestoDB</a>  Presto was designed and written<br>
from the ground up for interactive analytics and approaches the speed<br>
of commercial data warehouses while scaling to the size of<br>
organizations like Facebook.</li>
<li><a href="http://druid.io/">Apache Druid</a> : Druid is an is a high performance<br>
real-time analytics database</li>
<li><a href="https://pinot.apache.org/">Pinot</a> : Pinot is a realtime distributed<br>
OLAP datastore, which is used at LinkedIn to deliver scalable real<br>
time analytics with low latency.</li>
<li><a href="https://prometheus.io/">Prometheus</a> : It is used to record real-time<br>
metrics in a <a href="https://en.wikipedia.org/wiki/Time_series_database" title="Time
series database">time series<br>
database</a> (allowing for high dimensionality) built using a<br>
HTTP pull model, with flexible queries and real-time alerting.</li>
<li><a href="https://www.influxdata.com/">influxdb</a> : InfluxDB is the open source time series database</li>
</ul>
<h2 id="cicd">CI/CD</h2>
<ul>
<li><a href="https://www.spinnaker.io/">Spinnaker</a> : Spinnaker is an open source, multi-cloud continuous delivery platform for releasing software changes with high velocity and confidence.</li>
</ul>
<h2 id="machine-learning-and-ai">Machine Learning and AI</h2>
<ul>
<li><a href="https://pytorch.org/">PyTorch</a> : Deep learning Framework.</li>
<li><a href="https://m3db.io/">M3 db</a> : ## The fully open source metrics platform<br>
built on M3DB, a distributed timeseries database</li>
<li><a href="https://www.tensorflow.org/">Tensor Flow</a> : TensorFlow is an<br>
end-to-end open source platform for machine learning</li>
</ul>
<h2 id="all-about-containers">All About Containers</h2>
<ul>
<li><a href="https://eng.uber.com/peloton/">Peloton</a> : a unified scheduler<br>
designed to manage resources across distinct workloads, combining our<br>
separate clusters into a unified one</li>
<li><a href="http://mesos.apache.org/">Mesos</a> : Apache Mesos abstracts CPU, memory, storage, and other compute resources away from machines (physical or virtual), enabling fault-tolerant and elastic distributed systems to easily be built and run effectively.</li>
<li><a href="https://kubernetes.io/">Kubernetes</a> :  Automated container deployment, scaling, and management</li>
<li><a href="https://netflix.github.io/conductor/">Conductor</a> : Conductor is a microservices orchestration engine</li>
</ul>
<h2 id="data-analytics">Data Analytics</h2>
<ul>

interactive dashboards.</li>
</ul>
<h2 id="networking">Networking</h2>
<ul>
<li><a href="https://grpc.io/">grpc</a> : A high performance, open-source universal RPC framework</li>

</ul>
<h2 id="serverless">Serverless</h2>
<ul>

<li><a href="https://serverless.com/">Serverless Framework</a> : The Serverless Framework consists of an open source <strong><a href="https://serverless.com/framework/docs/providers/">CLI</a></strong> that makes it easy to develop, deploy and test serverless apps across different cloud providers.</li>
</ul>
<h2 id="testing">Testing</h2>
<ul>

output.</li>
<li><a href="https://taiko.gauge.org/">taiko</a> : <a href="https://github.com/getgauge/taiko">Taiko</a> is a free and open source browser automation tool built by the team behind <a href="https://gauge.org/">Gauge by ThoughtWorks</a>.</li>
</ul>
<h2 id="distributed-system">Distributed System</h2>

<h2 id="contributing">Contributing</h2>
<p>Open an issue first to discuss potential changes/additions. If you have questions with the guide, feel free to leave them as issues in the repository. If you find a typo, create a pull request. The idea is to keep the content up to date and use github’s native feature to help tell the story with issues and PR’s, which are all searchable via google. Why? Because odds are if you have a question, someone else does too! You can learn more here at about how to contribute.</p>
